# 機器人提示訊息與錯誤訊息檢查報告

## 📋 檢查範圍
已檢查所有 `.gs` 檔案中的使用者提示訊息和錯誤訊息

---

## ⚠️ 發現的問題

### 1. **引號使用不一致**
**位置**: `constants.gs` 第 84 行
```javascript
ERROR_INVALID_FORMAT: '⚠️ 指令格式錯誤，請輸入"!教學" 來查看系統指令',
```
**問題**: 混用中文引號「」和英文引號 ""
**建議**: 統一使用中文引號「」或英文引號 ""
**修正**: `'⚠️ 指令格式錯誤，請輸入「!教學」來查看系統指令'`

---

### 2. **缺少空格**
**位置**: `constants.gs` 第 108 行
```javascript
SETTING_UPDATED: (setting, value) => `✅ ${setting}已更新為 ${value}`,
```
**問題**: `${setting}` 和「已更新為」之間缺少空格
**建議**: 在設定名稱後加上空格，讓訊息更易讀
**修正**: `✅ ${setting} 已更新為 ${value}`

---

### 3. **缺少表情符號（風格不一致）**
**位置**: `liff_gs.gs` 多處
- 第 65 行: `'缺少必要欄位'` → 應為 `'⚠️ 缺少必要欄位'`
- 第 73 行: `'僅限群組管理員可開團'` → 應為 `'⚠️ 僅限群組管理員可開團'`
- 第 82 行: `'找不到指定的場館'` → 應為 `'⚠️ 找不到指定的場館'`
- 第 128 行: `'開團成功！'` → 應為 `'✅ 開團成功！'`
- 第 137 行: `'開團失敗：${error.message}'` → 應為 `'❌ 開團失敗：${error.message}'`

**問題**: 這些訊息缺少表情符號，與其他訊息的風格不一致
**建議**: 統一加上對應的表情符號（⚠️ 警告、✅ 成功、❌ 錯誤）

---

### 4. **缺少表情符號（錯誤訊息常數）**
**位置**: `constants.gs` 第 93 行
```javascript
INVALID_COMMAND: '指令格式錯誤，請輸入!教學 來查看系統指令',
```
**問題**: 缺少 ⚠️ 表情符號，與其他錯誤訊息不一致
**建議**: 加上 ⚠️ 表情符號
**修正**: `'⚠️ 指令格式錯誤，請輸入!教學 來查看系統指令'`

---

### 5. **中英文混用**
**位置**: `group.gs` 第 80 行
```javascript
return '⚠️ 只有 admin 才能移除管理員';
```
**問題**: 使用英文 "admin"，但其他訊息都是中文
**建議**: 統一使用中文「管理員」或「admin 管理員」
**修正選項**:
- `'⚠️ 只有 admin 管理員才能移除管理員'`
- `'⚠️ 只有最高權限管理員才能移除管理員'`

---

### 6. **用詞不一致**
**位置**: 多處
- `constants.gs` 第 87 行: 使用「用戶」
- `group.gs` 第 50 行: 使用「使用者」
- `constants.gs` 第 94 行: 使用「使用者」

**問題**: 「使用者」和「用戶」混用
**建議**: 統一使用「使用者」（較正式）或「用戶」（較親切）
**建議統一為**: 「使用者」

---

### 7. **訊息語氣可以更友善**
**位置**: `group.gs` 第 20 行
```javascript
return '⚠️ 本群組已經有管理員囉，如需加入請洽現有管理員協助 🙇';
```
**問題**: 語氣較為生硬
**建議**: 可以更友善一些
**修正**: `'⚠️ 本群組已有管理員，如需加入請聯繫現有管理員協助 🙇'`

---

### 8. **訊息可以更清楚**
**位置**: `event.gs` 第 88 行（程式碼邏輯問題，非訊息問題）
```javascript
const totalParticipants = additionalCount == 0 ? additionalCount + 1 : additionalCount;
```
**問題**: 邏輯錯誤，應該是 `additionalCount == 0 ? 1 : additionalCount + 1`
**注意**: 這是程式碼邏輯問題，不是訊息問題，但建議一併修正

---

### 9. **錯誤訊息可以更具體**
**位置**: `liff_gs.gs` 第 65 行
```javascript
message: '缺少必要欄位'
```
**問題**: 沒有說明缺少哪些欄位
**建議**: 列出缺少的欄位，例如：`'⚠️ 缺少必要欄位：請確認已填寫日期、時間、場館'`

---

### 10. **訊息格式可以更一致**
**位置**: `event.gs` 第 163 行
```javascript
if (eventRegs.length === 0) return `📭 目前尚無人報名活動 ${eventCode}。`;
```
**問題**: 句尾有句號，但其他類似訊息沒有
**建議**: 統一格式，建議移除句號或統一加上句號

---

## ✅ 建議的修正優先順序

### 🔴 高優先級（影響使用者體驗）
1. **缺少表情符號** (`liff_gs.gs`, `constants.gs`) - 影響視覺一致性
2. **缺少空格** (`constants.gs` 第 108 行) - 影響可讀性
3. **引號不一致** (`constants.gs` 第 84 行) - 影響專業度

### 🟡 中優先級（改善體驗）
4. **中英文混用** (`group.gs` 第 80 行) - 統一語言風格
5. **用詞不一致** (多處) - 統一術語
6. **錯誤訊息不夠具體** (`liff_gs.gs` 第 65 行) - 幫助使用者理解問題

### 🟢 低優先級（優化建議）
7. **語氣優化** (`group.gs` 第 20 行) - 讓訊息更友善
8. **格式統一** (`event.gs` 第 163 行) - 統一標點符號使用

---

## 📝 總結

總共發現 **10 個問題**，主要集中在：
- 表情符號使用不一致（5 處）
- 用詞/語言不統一（3 處）
- 格式細節問題（2 處）

建議優先修正高優先級問題，以提升使用者體驗的一致性。
